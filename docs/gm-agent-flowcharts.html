建议并行化架构

只并行没有硬依赖的读取和事后处理。规则解析、世界提交、叙事仍保持顺序。

结论：并行化优先改造“读取长期记忆 / 检索内容”和“质检守卫 / 记忆候选提取”两处。 这两处不会改变规则解析、世界状态提交和叙事生成的事实顺序，因此质量风险最低。
